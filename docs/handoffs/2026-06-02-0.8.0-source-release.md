## Release theme

`0.8.0` is the evidence-to-repair foundation release.

The release makes runtime completeness explicit, makes public repair packets
fail closed unless they satisfy the actionability contract, routes static
limitations into analyzer backlog, and exposes readiness/attempt surfaces
without claiming autonomous repair.

## Report proof notes

Local Lane 1 report generation on the source release candidate produced
explicit limited runtime states where the local run was incomplete. The
Markdown surfaces preserved `Run status: limited_timeout` and
`Downstream consumable: false` rather than representing sampled or limited
output as full.

That limited state is acceptable for the release because 0.8.0 requires honest
runtime status, not a fake full-run claim.

## Non-claims

0.8.0 does not claim:

- autonomous code editing;
- provider integration;
- mutation execution;
- generated tests;
- default blocking CI gate semantics;
- default public badge semantic changes;
- complete full-repo analysis in every local environment;
- that limited or sampled reports are full runs;
- that every static signal is actionable;
- that static limitations are repair packets.
