full run.

## Recommended next action

In source `ripr`:

```powershell
cd <source-ripr-checkout>
rtk git status --short --branch
rtk git fetch origin --prune
rtk git remote add swarm git@github.com:EffortlessMetrics/ripr-swarm.git
rtk git fetch swarm --prune
rtk git switch -c release/0.8.0-swarm-sync origin/main
rtk git log --oneline --left-right --cherry-pick origin/main...swarm/main
rtk git diff --stat origin/main..swarm/main
rtk git merge --no-ff --no-commit swarm/main
```

If histories look unrelated, stop and inspect the remote/repo root. Do not use
`--allow-unrelated-histories` blindly.

Commit the source sync as:

```text
release: sync ripr-swarm for 0.8.0
```

Then run source validation serially and open the source release PR with the
validation evidence.

## What not to do

- Do not start another analyzer slice before source sync.
- Do not merge source `ripr` with a squash import.
- Do not import local artifacts, target cache, rescue patches, or agent scratch
  files into source.
- Do not weaken release, signing, publishing, or distribution surfaces from
  source `ripr`.
- Do not present queued, pending, failed, or missing CI as green.
