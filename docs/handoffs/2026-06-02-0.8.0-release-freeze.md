## Source Sync Boundary

The source `ripr` sync should use a merge commit from `ripr-swarm` into source
`ripr`. Do not squash this development history into one source commit.

Recommended source merge commit message:

```text
release: sync ripr-swarm for 0.8.0
```

Authority during source conflict resolution:

- source `ripr` wins for release, publishing, signing, workflow, marketplace,
  distribution metadata, and source branch protection assumptions;
- `ripr-swarm` wins for analyzer/report/runtime code, Lane 1 schemas/specs,
  fixtures, xtask report behavior, repair-loop/readiness outputs, capability
  updates, and traceability updates.

## Release Blockers

Block source release prep on:

- dirty source sync worktree;
- merge conflicts in release, signing, publishing, or distribution surfaces not
  reviewed;
- failing required source checks;
- `Cargo.toml` / `Cargo.lock` version drift;
- schema docs contradicting generated outputs;
- release notes overstating product claims;
- limited or sampled report output represented as a full run;
- local artifacts, target cache, rescue patches, or agent scratch files entering
  the source PR.

Do not block on:

- zero actionable packets in a sampled run, if limitations are explicit;
- high static limitation counts, if routed and named;
- advisory Droid review infrastructure noise when required source checks pass;
- lack of provider integration;
- lack of autonomous repair.

## Non-Claims

0.8.0 must not claim:

- autonomous code editing;
- provider integration;
- mutation execution;
- default blocking CI or badge semantic switch;
- every repository run is full;
- static limitations are safe repair packets.

Limited and sampled evidence must stay explicit and must not be presented as a
full run.
