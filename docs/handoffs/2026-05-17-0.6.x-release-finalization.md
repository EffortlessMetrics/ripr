## Release Decision

Do not ship yet.

Ship only after the named blockers below are either resolved or explicitly
accepted by the maintainer:

1. The open PR board needs an explicit 0.6.x disposition pass. Current triage
   scanned 35 open PRs and reported advisory `status: warn`; most branches are
   behind, several are conflicted or failing, and some open PRs plausibly touch
   release-critical diff parsing or analyzer correctness.
2. The external-adopter first-run smoke proves the installed binary can find
   and repair one Rust gap outside `ripr`, but it also exposed a start-here
   integration boundary: a repo-exposure-derived gap ledger is repo-scoped, so
   `ripr first-pr` correctly reports `no_action` instead of selecting that gap
   as PR-local. The pilot summary still names the repairable gap. Maintainers
   should decide whether this is acceptable for 0.6.x or whether the release
   needs a PR-local first-pr proof before tag.
3. A final release proof should be rerun after the board disposition decision,
   because `release-readiness` currently passes required checks with a
   non-required `repo-exposure-latency` warning.
