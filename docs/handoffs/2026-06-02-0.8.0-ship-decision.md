0.8.0 public copy may claim:

- runtime completeness is explicit across Lane 1 reports;
- public actionability projection is stricter and requires actionable gap state;
- incomplete repair packets fail closed instead of becoming swarm-ready work;
- readiness exposes blocked states, missing fields, and limitation routes;
- repair packets carry proof and edit-boundary expectations when they satisfy
  the public packet contract;
- static limitations are routed into analyzer backlog instead of fake
  actionability;
- cache report and GC support improve report operability;
- the source release preserves release/distribution authority in `ripr` while
  `ripr-swarm` remains the development trunk.

0.8.0 public copy must not claim:

- autonomous repair;
- source-edit automation;
- generated tests;
- provider or model calls;
- runtime mutation execution;
- killed or survived mutation outcomes;
- runtime adequacy;
- coverage adequacy;
- correctness proof;
- default CI blocking;
- public badge semantic switch;

- TypeScript, JavaScript, or Python preview promotion beyond documented
  advisory status.

crate publication, server assets, marketplace surfaces, badge semantics, and


