## Allowed Before Tag

Only these changes may land before the 0.6.x tag without reopening the release
scope:

- a docs typo or public-copy correction that fixes a false release claim;
- a packaging failure fix found by the final proof;
- a `cargo xtask release-readiness --version 0.6.0` failure fix;
- a security, secret, or credential exposure fix;
- a hard CI failure on the freeze base;
- `#1210`, but only after explicit maintainer approval that the
  activation-value burn-down is part of 0.6.x.

Every allowed change must rerun the affected proof commands and update the
release handoff chain if it changes the candidate base.

## Not Allowed Before Tag

Do not land these classes as part of 0.6.x finalization:

- broad refactors;
- analyzer improvements that are not named release blockers;
- preview-language promotion;
- new public CLI, CI, editor, policy, or report surfaces;
- new generated-evidence behavior;
- new policy authority;
- default CI blocking;
- branch protection changes;
- baseline adoption;
- suppression creation or semantic changes;
- generated tests;
- provider calls;
- mutation execution;
- badge endpoint refreshes.

These can resume after the 0.6.x tag or in a separately approved release
candidate.
